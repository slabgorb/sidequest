# Story 53-1: RigComposurePool — extend Edge framework for vessel-attached pools

## Objective
Extend the EdgePool framework (ADR-014, ADR-078) to support vessel-attached pools. RigComposurePool tracks the structural composure of a rig (vehicle) that is bound to a character. This story is the foundation for Epic 53 (Road Warrior).

## Acceptance Criteria
- [ ] RigComposurePool class defined in `sidequest/game/pools.py` with the following:
  - Inherits from or mirrors EdgePool pattern
  - Tracks `composure` integer (max and current)
  - Bound to a `character_id` and `rig_item_id`
  - Supports delta operations (gain/lose composure)
  - Zero-crossing detection (Composure→0 triggers event, handled in 53-3)
  - Serialization to/from character state
- [ ] Type contract in `sidequest/protocol/` (if needed for WebSocket/REST)
- [ ] Unit tests in `tests/game/test_pools.py`:
  - Initialization, max/current bounds
  - Delta operations (gain/lose)
  - State serialization round-trip
  - Zero-crossing detection (no-op side effects; 53-3 handles crash)
- [ ] OTEL integration stub (53-4 completes this):
  - Import `TraceProvider` and define spans (rig_pool.created, rig_pool.delta, rig_pool.zero_crossing)
  - Emit in RigComposurePool methods
- [ ] Wiring test: RigComposurePool is imported and reachable from game state module (integration test per CLAUDE.md principle)
- [ ] No unimplemented crash logic (Composure→0 fires in 53-3; this story just detects it)

## Design Notes
- **Pattern match**: EdgePool (ADR-078) for API, composure semantics (ADR-078 + ADR-024 dual-track tension model)
- **Binding model**: vessel_item_id + character_id unique pair; one rig per character per session (can swap vessels)
- **Composure→0 is NOT auto-applied damage**: RigComposurePool stores the pool; the crash handler (53-3) decides injury tags + Edge loss
- **OTEL observability**: Per ADR-031 and CLAUDE.md OTEL principle, every subsystem decision must emit watcher events. Stub in place; 53-4 surfaces to GM panel.

## Technical Context
- **Parent class**: Likely inherits from or mirrors `sidequest/game/pools.py:EdgePool`
- **Storage**: Part of character state; serialized in session persistence (53-2 materializer integration updates this)
- **Mutation API**: `apply_composure_delta(delta: int) -> int` (new value), `is_destroyed() -> bool` (zero-crossing)
- **References**: 
  - ADR-014 Diamonds and Coal
  - ADR-078 Edge / Composure Combat
  - ADR-031 Game Watcher (for OTEL pattern)
