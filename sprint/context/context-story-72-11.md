## Problem
BUG (NPC Identity Hardening; surfaced by 71-31, coyote_star). pregen.seed_manual silently drops world cultures beyond the first four. sidequest/server/dispatch/pregen.py:225 slices the resolved culture list — cultures = [c.name for c in effective[:MAX_CULTURES]] — where MAX_CULTURES=4 (pregen.py:48) is a Rust-port artifact (".take(4)"), NOT a design decision. effective_cultures(world) correctly resolves coyote_star's 5 world cultures (broken_drift, free_miners, hegemonic, tsveri, voidborn); the [:4] slice then discards the 5th. The drop is ORDER-DEPENDENT — whichever culture sorts last dies (voidborn, alphabetically) — so zero voidborn NPCs ever seed. Two CLAUDE.md violations: (1) No Silent Fallbacks — a configured culture is dropped with no log/signal; (2) the OTEL lie-detector is blind to its own lie: the pregen.seed_manual span (pregen.py:317-330) emits culture_count=len(cultures) POST-cap (4), so the GM panel reports "4 cultures" and looks healthy while voidborn is missing.

DESIGN DIRECTION (Architect / White Queen, 2026-06-02, reuse-first, no new infra, no ADR): (1) DELETE the [:MAX_CULTURES] slice — seed ALL effective cultures; npc_count = len(cultures)*NPCS_PER_CULTURE (pregen.py:229) scales on its own (coyote_star -> 5x3=15 NPCs incl voidborn). (2) REMOVE the now-dead MAX_CULTURES constant (pregen.py:48) — dead code is worse than no code. Do NOT replace it with a smaller random sample and do NOT merely bump it to 5 (that "passes" coyote_star while still silently breaking the next 6-culture world). The world author's declared culture list IS the bound (Crunch in Genre, Flavor in World). (3) Make the span tell the truth: add effective_culture_count (pre-seed, the world's true count) alongside the seeded count; for coyote_star both read 5 with cultures_source=world — that triple is the proof voidborn seeded. Any FUTURE culture drop emits a WARNING + span flag, never a silent slice. SEAM: one file (sidequest/server/dispatch/pregen.py) + span attrs in telemetry/spans/pregen.py; test extends tests/server/dispatch/test_pregen.py. No content change (coyote_star already declares voidborn). Repos: sidequest-server.
