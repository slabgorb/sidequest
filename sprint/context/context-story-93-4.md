## Problem
Wire the character-sheet History section to player-linked lore. The History section (93-3) is designed as the home for player-linked lore; this story attaches it to the existing lore store. Story 75-15 already persists creation-seed lore fragments across resume (rag.character_creation_lore_seeded) via the ADR-048 lore RAG store — surface the fragments that are linked to THIS character/player in the History section beneath the origin block. 

Server: expose the player-/character-linked lore fragments in (or alongside) the character snapshot — a typed list of {fragment_id, title/summary, source} filtered to the character's creation-seed + player-attributed fragments (perception/visibility honored). 

UI: render those as a 'Lore' subsection under History, linking to the lore page where one exists. 

No new lore authoring — this is plumbing existing fragments to the surface. 

**NO SILENT FALLBACK:** if a linked fragment id resolves to nothing, log/skip loudly, don't fabricate.

Refs: ADR-048 lore RAG; story 75-15 (creation-seed lore persistence); depends on 93-3 (History section) + 93-2 (creation_answers).

## Acceptance Criteria
- Server exposes a typed list of player-/character-linked lore fragments (creation-seed + player-attributed) in the character snapshot, filtered by perception/visibility.
- The History section renders a 'Lore' subsection under the origin block listing those fragments (title/summary), linking to the lore page where a route exists.
- Fragments are sourced from the existing ADR-048 store (75-15 creation-seed persistence) — no new authoring; a fresh session's seeded fragments appear after creation.
- An unresolvable linked fragment id is skipped loudly (logged), never fabricated (No Silent Fallbacks).
- OTEL/observability: emit a span/log for lore-link resolution count per character so the GM panel can confirm fragments reached the surface.
- Wiring + perception test: a character's linked fragments reach the snapshot and render; another player's private fragments do NOT leak into this character's History.
