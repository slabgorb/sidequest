Durable creation_answers provenance model + snapshot exposure

## Problem
Stop orphaning the player's chargen answers. Today freeform text is consumed only for the {class} prose slot then discarded, and selections survive only as flavor labels (origin_label/calling_label/background/drive) — the per-scene answer the player actually gave is never durably recorded. Add a creation_answers provenance record to the Character model: an ordered list, one entry per answered chargen scene, each carrying scene_id, prompt (the scene's question/title), kind ('choice' | 'freeform'), value (the player's verbatim freeform text OR the chosen option label), and an optional archetype_inferred flag/marker for the scene(s) whose freeform fed 93-1's inference. Populate it in builder.build() from self._results (ChoiceInput.choice_label / FreeformInput.text are already captured on SceneResult — this is wiring, not new capture). Expose creation_answers in the character snapshot (CharacterSheetDetails / views.py serialization) so the UI can read it. Server-only; the UI render is 93-3. Refs: builder.py ~1730 (apply_choice/choice_label), ~1790 (apply_freeform/FreeformInput), ~2654 (build); character.py model; server/views.py snapshot.

## Acceptance Criteria
- Character gains a typed creation_answers field: ordered list of {scene_id, prompt, kind: choice|freeform, value, archetype_inferred?}.
- After an all-freeform build, creation_answers contains one entry per answered scene with kind='freeform' and value = the player's verbatim text (not a derived label).
- After a preset build, each entry has kind='choice' and value = the chosen option label; combined-origin and drive scenes are represented faithfully.
- creation_answers is carried in the character snapshot payload and round-trips through save/load (persisted, not recomputed).
- The scene(s) whose freeform fed 93-1's archetype inference are marked archetype_inferred=true so the UI can badge them.
- Wiring test: a snapshot built from a real chargen flow exposes creation_answers (asserts it reaches the serialized payload, not just the in-memory model).
