## Problem
Make the perseus_cloud orrery a LOCAL map that follows the party's location node, instead of a single world-wide chart. Two halves: (1) the authored sector-tree content (genre_packs/space_opera/worlds/perseus_cloud/orbits.yaml — 140 bodies, one parent-less PERSEUS CLOUD hub -> 34 system stars -> planets/belts/stations -> moons; geometry synthesized via Kepler since the sector export carries none) — ALREADY MERGED in sidequest-content#383; (2) bind region->orbital scope so the chart auto-centers on the party's current system. KEY FACT: star body-ids equal cartography region ids by construction (identity join, e.g. region 'yula' -> star body 'yula'). Region-mode location truth is snapshot.pc_regions[player_name], mutated via WorldStatePatch(pc_region=...) at agents/subsystems/movement.py:367 and server/narration_apply.py:3164; session-bind path is server/session_room.py bind_world; scope state is server/session.py orbital_scope + snapshot.party_body_id. DESIGN DECISION (v1): pc_regions is per-PC but party_body_id is singular — v1 binds to the ACTING/seated PC's region on each pc_region change (correct for a co-located party); stricter MP cross-system co-location handling is deferred. OUT OF SCOPE: 'set a course' travel mechanics, chart.yaml flavor layer (pending current style direction). Refs: ADR-094 (orrery), ADR-130 (orbital clock), CLAUDE.md OTEL + No-Silent-Fallbacks.
