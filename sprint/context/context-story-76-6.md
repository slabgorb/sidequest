Source coverage: sync stateful snapshot.npcs into the entity index — add an Npc projector path (project_npc_card currently takes only NpcPoolMember) and extend sync_entity_cards so scene-stateful NPCs, not just pool members, are retrievable.

## Problem
The entity index (ADR-118: Universal Retrieval Layer) currently only syncs NPC pool members via `project_npc_card(NpcPoolMember)`. Scene-stateful NPCs (`snapshot.npcs`) — those created within a session's game state — are not indexed and therefore not retrievable via `_retrieve_entities_for_turn()`. This makes the "universal" retrieval index incomplete.

To unblock the full value of story 75-7 (OTEL/GM-panel) and 75-8 (e2e testing), we need to extend the sync and projection layer to include snapshot NPCs alongside pool members.

## Technical Approach
1. Extend `project_npc_card()` to accept both `NpcPoolMember` and stateful NPC instances from `snapshot.npcs`.
2. Update `sync_entity_cards()` in the entity index worker to iterate over `snapshot.npcs` and call the extended projector.
3. Ensure the projector can extract the necessary NPC attributes (name, description, traits, disposition) from both pool and snapshot sources.
4. Add test coverage for sync_entity_cards with scene-stateful NPCs (including edge cases: empty snapshot, mixed pool+snapshot scenes).

## Scope
- In scope: projector path extension, sync worker update, snapshot NPC indexing.
- Out of scope: factions/locations (story 76-7), embed telemetry (story 76-5), test helper consolidation (story 76-9).

## Acceptance Criteria
1. `project_npc_card()` can receive both NpcPoolMember and snapshot NPC instances and returns a valid card for both.
2. `sync_entity_cards()` iterates over `snapshot.npcs` and projects each via the extended `project_npc_card()`.
3. Test coverage: passing test that verifies snapshot NPCs are indexed and retrievable via entity query. Include edge case: a scene with both pool and snapshot NPCs.
4. Full server suite green (pytest tests/); ruff check + format clean.
