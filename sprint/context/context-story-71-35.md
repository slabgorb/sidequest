## Problem
Playtest 2026-05-31 (burning_peace MP) + Keith: the 'GM seat' is a category error. SideQuest's thesis is *the narrator is the GM; every human is a player* — no human occupies a GM seat. The is_gm() predicate + the CoreInvariantStage gm-sees-all short-circuit were ported from tabletop-VTT convention and don't fit.

KEY FACT — it's already always-false dead code: gm_player_id is initialized None in views.py and NEVER reassigned (the MP branch only logs a warning; it never names a GM). view.py is_gm() = 'gm_player_id is not None and …' → always False in every mode. So the gm_sees_all short-circuit at invariants.py is already dead, never taken. 'Wire it false' is a no-op; the real defect is the dead axis still existing (a dormant footgun inviting a future 'seat a GM' PR, and dead code which CLAUDE.md forbids). grep of every genre pack for 'unless: is_gm()' = 0 matches.

DELETE the axis (behaviour-preserving), don't wire it false:
- remove the gm_sees_all branch (game/projection/invariants.py — the 'if view.is_gm(player_id)' block + its source='invariant:gm_sees_all')
- remove _is_gm + its 'is_gm' registry entry (game/projection/predicates.py)
- remove is_gm from the GameStateView Protocol + SessionGameStateView impl + the gm_player_id field (game/projection/view.py)
- remove the gm_identity_unwired warning + GM-player TODO + the _gm_wiring_warned flag + the gm_player_id local (server/views.py:~115-169,262-263)
The narrator gets canonical state because it is server-side and NOT a projection recipient at all — not because it is 'the GM' in the filter. The firewall then stands on player-identity predicates alone (is_self/is_owner_of/in_same_zone/visible_to/in_same_party).
